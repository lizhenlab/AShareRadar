# Requirements Specification

## 1. Purpose

AShareRadar provides a local, Chinese-language A-share single-stock research workbench for personal study. It helps a user inspect quote state, trend structure, data quality, risk signals, alerts, notes, and research evidence for one stock at a time.

The product intentionally avoids portfolio optimization, automated trading, account login, and order execution.

## 2. Stakeholders

- Primary user: an individual A-share investor or researcher.
- Maintainer: future developer extending providers, analysis logic, UI panels, tests, and documents.
- External systems: Tencent/Eastmoney public endpoints, AKShare, BaoStock, Tushare, optional Futu OpenAPI, optional OpenAI-compatible LLM API.

## 3. Product Scope

### In Scope

- Query one A-share stock by 6-digit code or standard symbol, or discover it by Chinese name in the main-query and watchlist inputs.
- Load a workbench snapshot containing quote, daily K-lines, data quality, trend score, support/resistance, advice, insights, chart marks, alerts, alert events, and notes, with companion advice-timeline, minute-analysis, exact chart-inspection, and local research-activity views.
- Display market overview and sample strong-stock observations.
- Maintain a local watchlist research queue, conclusion snapshots and timeline, snapshot-bound review plans/evaluation history, current or custom historical scans, alerts, alert events, notes, chart marks, monitoring events, task runs, provider state, and cached market data.
- Export/import user-owned data with guarded rollback, and inspect or run bounded local-data retention cleanup.
- Explain analysis through transparent rule logic, with optional LLM wording enhancement.
- Provide diagnostics for cache freshness, provider health, scheduler state, and storage size.

### Out of Scope

- Brokerage account integration.
- Automated order placement.
- Portfolio-level allocation.
- Guaranteed real-time market data.
- Financial, legal, or investment advice.
- Licensed announcement/news calendars, formal report-period financial trends, and multi-stock aggregation/comparison.

## 6. Data Requirements

- Quote fields: code, market, name, price, previous close, open, high, low, volume, amount, change, change percent, optional turnover, PE, PB, market cap, source, and a market event timestamp normalized to fixed `Asia/Shanghai` `YYYY-MM-DD HH:MM:SS` text.
- K-line fields: date/timestamp, OHLC, volume, optional amount, turnover, interval, source, cache metadata; minute report status also carries `availability`, `availability_reason`, `reason_code`, and audit `klines`.
- User data: watchlist queue metadata and comparable-change unread/view state, versioned advice conclusion snapshots and timeline comparisons, snapshot-bound review plans and evaluation history, alert rules, alert events, and notes. Viewed-through advice IDs are request watermarks, not a separate persisted history table. The local research-activity feed is a browser-side projection of existing records and has no separate persistence model.
- Runtime data: provider status, provider capability status, task runs, monitor events, cache events, trading calendar, full-market scan runs/results, retry lineage, per-result quote/K-line/metadata degradation booleans, and normalized degradation-reason codes.
- Optional remote LLM context: current question/topic, symbol/name, deterministic rule answer, authoritative conclusion/confidence/support/resistance/actions/invalidations, selected quote/indicator facts, up to four data-quality notes, and up to six evidence items. Local user-state collections and provider credentials are excluded; the LLM API key is sent only as transport authentication to the configured endpoint.

## 7. External Provider Requirements

- Tencent/Eastmoney style public endpoints are best-effort and can change. Tencent daily K-lines use the current SH/SZ/BJ `newfqkline` interface.
- The AKShare daily adapter falls back to Eastmoney direct access and then to a rate-limited Sina raw-daily plus safely parsed forward-adjustment-factor path; the Sina path is a last resort, not an independent primary provider.
- Full-market orchestration must classify a complete quote/daily provider-chain outage separately from stock-level no-data coverage so retryable pending work is not persisted as bulk `missing` results.
- AKShare, BaoStock, Tushare, and Futu are optional dependencies configured through installed packages and environment variables.
- Tushare requires `ASHARE_RADAR_TUSHARE_TOKEN`; `TUSHARE_TOKEN` is accepted as a legacy alias.
- Futu requires `ASHARE_RADAR_FUTU_ENABLED=1` and local OpenD availability.
- The app must never silently treat demo data as real data unless `ASHARE_RADAR_DEMO_PROVIDER_ENABLED=1`.
