# 国内同类产品调研：个股研究工作流

> 调研对象：东方财富、同花顺、雪球、富途牛牛
> 检索日期：2026-07-15
> 目标产品：AShareRadar，本地、单用户、A 股个股研究工作台
> 证据原则：优先采用产品官网、帮助中心、开发者发布的官方应用说明；营销文案仅用于确认产品设计，不视为效果背书。

## 1. 结论摘要

四类产品已经形成相似的闭环：从行情或资讯发现股票，进入个股页查看图表、资金与基本面，结合事件和社区/AI解释形成判断，再通过自选与预警持续跟踪；具备券商能力的产品会把流程延伸到开户和下单。

它们的差异主要在研究入口和商业边界：

- 东方财富以行情、F10、数据中心和资金数据为骨架，叠加妙想 AI，并把 Level-2、Choice 与证券交易作为扩展层。
- 同花顺以图表技术分析、问财条件选股、Level-2/DDE 和丰富预警见长，高级诊股与行情权益分层明显。
- 雪球以社区内容、组合和事件讨论驱动研究，行情与交易服务形成闭环；AI 正被用于搜索与财报内容总结。
- 富途以统一个股页、精细图表、资金/筹码、事件预警和跨端工作区见长，但真实交易能力受券商账户、市场和地域资格约束。

对 AShareRadar 最值得借鉴的不是功能数量，而是四项结构能力：

1. 以个股为中心，把行情、图表、事件、资金、基本面、风险和笔记放进同一研究上下文。
2. 所有结论都能回到原始证据、数据时间和计算口径。
3. 自选、事件与预警组成持续跟踪闭环，并提供去重、冷却、确认和恢复状态。
4. AI 只做带引用的解释与总结；数据不足时明确拒答，不进入推荐和自动交易。

## 8. 共性核心流程

### 8.1 发现到纳入跟踪

榜单、热点、资讯或自然语言选股 → 查看筛选条件与结果 → 进入个股页 → 加入自选/动态分组。

这条流程在东方财富数据选股、同花顺问财、雪球筛选器和富途多因子选股中都成立。区别在于结果是否能解释：同花顺把自然语言映射为条件，富途展示因子分类，东方财富依赖数据中心，雪球更偏基础分类。

证据：[东方财富选股器](https://data.eastmoney.com/xuangu/)、[问财选股](https://www.iwencai.com/stockpick/index?allow_redirect=false)、[雪球选股器说明](https://xueqiu.com/edu/invest-edu/simu/)、[富途选股器帮助](https://support.futunn.com/topic105)（检索：2026-07-15）。

### 8.2 个股证据下钻

自选或搜索 → 个股行情与状态标签 → K 线/技术指标 → 资金与筹码 → 财务/F10 → 公告、新闻和事件 → 原文。

成熟产品都把这些信息留在同一个股票上下文，降低重复搜索成本。资金和筹码的口径并不一致，因此“看起来相同的指标”不能跨产品直接比较。

证据：[东方财富金融终端功能](https://emdesk.eastmoney.com/pc_activity/Pages/VIPTrade/pages/link/gnjs.html)、[同花顺 K 线帮助](https://www.10jqka.com.cn/ad_mar/man/05-2.htm)、[富途股票分析帮助目录](https://support.futunn.com/categories/2197?from_platform=1&lang=zh-cn)（检索：2026-07-15）。

### 8.3 事件触发再研究

自选/持仓 → 价格、量价、技术、异动或公告条件 → 系统通知 → 回到触发时刻和个股上下文 → 查看原始事件与最新行情 → 更新判断。

富途把提醒类型和频率公开得最清晰；同花顺与东方财富则强调智能盯盘、自选公告和热点订阅；雪球用股价与持仓公告提醒完成基础闭环。

证据：[富途个股提醒](https://support.futunn.com/topic597?from_platform=1&lang=zh-cn)、[同花顺 App 官方说明](https://apps.apple.com/cn/app/%E5%90%8C%E8%8A%B1%E9%A1%BA-%E7%82%92%E8%82%A1-%E8%82%A1%E7%A5%A8/id303191318)、[东方财富 App 官方说明](https://apps.apple.com/cn/app/%E4%B8%9C%E6%96%B9%E8%B4%A2%E5%AF%8C-%E8%82%A1%E7%A5%A8-%E7%82%92%E8%82%A1-%E7%90%86%E8%B4%A2/id423525686)、[雪球自选与提醒 FAQ](https://xueqiu.com/about/faq/3/1)（检索：2026-07-15）。

### 8.4 AI 摘要到原始证据

提出问题或打开个股摘要 → AI 汇总关键变化 → 展示引用、时间与风险 → 点击回到公告/研报/新闻原文 → 用户自行判断。

东方财富妙想强调问答和研报摘要可追溯来源；雪球近期版本强化引用展示；富途明确模型可能出错，并在没有显著信息时不生成解读。这比只给“看多/看空”结论更适合研究工具。

## 11. 建议的最小实现顺序

1. P0：统一个股上下文、数据质量/权限标签、事件时间线。
2. P0：本地预警状态机，包括触发、冷却、确认、恢复和证据快照。
3. P0：AI 回答契约与规则回退，所有回答可回到原始来源。
4. P0：资金/筹码口径卡片，明确观测、推导、估算和缺失。
5. P1：可解释筛选、动态自选和注意力队列。
6. P1：盘前/盘后/周度变化摘要与风险优先区。
7. P2：少量图表标注和本地导入导出。

每一步都应先完成数据时间、来源、授权状态和失败降级，再增加新的分析指标。这样可以保留成熟产品的研究闭环，同时避开交易、社区、云账号和会员体系带来的范围膨胀。
